TASK 1: Data Ingestion — Fund Holdings

Create: backend/jobs/sync_holdings.py

This script fetches mutual fund holdings from AMFI monthly portfolio disclosures.

AMFI publishes portfolio disclosures at:
https://www.amfiindia.com/research-information/other-data/scheme-portfolio

The data is in CSV/text format per AMC, published by the 10th of every month 
for the previous month.

ALTERNATIVE (use this for MVP — faster): 
Use the mfapi.in API which provides NAV history.
For holdings, AMFI publishes them in a structured format.

For the MVP, use a simpler approach:
1. Fetch the AMFI portfolio disclosure text for the most recent month
2. It's a large text file with this approximate format:
   Fund Name: HDFC Equity Fund
   Portfolio as on: 31-Mar-2025
   Company Name | ISIN | Industry | Quantity | Market Value (Rs. Lakhs) | % to NAV
   
3. Parse this into fund_holdings table entries
4. Store with disclosure_date

NOTE: AMFI portfolio disclosures can be inconsistent in format.
Build the parser to be fault-tolerant — skip lines that don't parse, 
log the errors, continue processing.

Also create a MANUAL HOLDINGS SEEDER for testing:
Create a Python script that seeds test holdings for 10 popular funds 
(HDFC Equity, Parag Parikh Flexi Cap, Mirae Asset Large Cap, SBI Bluechip,

This gives you REAL data to test overlap with immediately.

TASK 2: Overlap Engine

Create: backend/services/overlap_engine.py

class OverlapEngine:
    
    def compute_overlap(
        self, 
        fund_holdings: Dict[str, List[Holding]]  # scheme_code → holdings list
    ) -> OverlapResult:
        """
        Core logic:
        1. For each pair of funds (A, B), find common stocks
        2. Calculate overlap % = sum of min(% in A, % in B) for each common stock
        3. Return pairwise overlap matrix
        
        Example:
        Fund A holds Reliance at 8%, Infosys at 6%
        Fund B holds Reliance at 5%, TCS at 7%
        Common stocks: Reliance
        Overlap score = min(8, 5) = 5%
        """
    
    def compute_sector_exposure(
        self,
        fund_holdings: Dict[str, List[Holding]],
        weights: Dict[str, float]  # scheme_code → allocation weight (0-1)
    ) -> Dict[str, float]:
        """
        Aggregate sector exposure across all funds weighted by allocation.
        Returns: {"IT": 28.5, "Banking": 22.1, "FMCG": 15.3, ...}
        """
    
    def compute_stock_concentration(
        self,
        fund_holdings: Dict[str, List[Holding]],
        weights: Dict[str, float]
    ) -> List[StockExposure]:
        """
        For each stock, calculate total portfolio exposure across all funds.
        Returns sorted list: [{"stock": "Reliance", "exposure": 4.2, "funds": ["HDFC", "Parag Parikh"]}, ...]
        """
    
    def compute_marketcap_breakdown(
        self,
        fund_holdings: Dict[str, List[Holding]],
        weights: Dict[str, float]
    ) -> Dict[str, float]:
        """
        Returns: {"Large Cap": 65.2, "Mid Cap": 22.1, "Small Cap": 8.4, "Debt/Cash": 4.3}
        Classify stocks by market cap using NSE classification or AUM-based heuristic.
        """

Data models for this service (in schemas.py):

class Holding(BaseModel):
    stock_isin: str | None
    stock_name: str
    sector: str | None
    holding_percentage: float
    asset_type: str  # Equity/Debt/Cash

class PortfolioFund(BaseModel):
    scheme_code: str
    scheme_name: str
    units: float
    nav: float
    current_value: float
    expense_ratio: float | None
    plan_type: str  # Direct/Regular

class OverlapPair(BaseModel):
    fund_a_code: str
    fund_a_name: str
    fund_b_code: str
    fund_b_name: str
    overlap_score: float  # 0-100
    common_stocks: List[str]
    common_stock_count: int

class AnalysisResult(BaseModel):
    session_id: str
    funds: List[PortfolioFund]
    total_value: float
    overlap_matrix: List[OverlapPair]
    sector_exposure: Dict[str, float]
    marketcap_breakdown: Dict[str, float]
    top_stock_concentrations: List[dict]
    expense_audit: ExpenseAudit
    health_score: int  # 0-100
    health_explanation: str  # LLM generated
    red_flags: List[str]
    created_at: datetime

class ExpenseAudit(BaseModel):
    total_weighted_expense_ratio: float
    regular_plan_funds: List[str]
    potential_savings_yearly: float
    benchmark_expense_ratio: float  # industry avg for this category mix

TASK 3: Health Scorer

Create: backend/services/health_scorer.py

The health score is calculated from these weighted components:

1. OVERLAP SCORE (30% weight): 
   - Score 100 if max pairwise overlap < 10%
   - Score 70 if max overlap 10-20%
   - Score 40 if max overlap 20-35%
   - Score 10 if max overlap > 35%

2. EXPENSE RATIO SCORE (20% weight):
   - Score 100 if weighted expense ratio < 0.5%
   - Score 80 if 0.5–0.8%
   - Score 60 if 0.8–1.2%
   - Score 30 if 1.2–1.8%
   - Score 0 if > 1.8%

3. DIRECT PLAN SCORE (15% weight):
   - Score 100 if all funds are Direct plans
   - Deduct 20 points per Regular plan fund

4. CONCENTRATION SCORE (20% weight):
   - Score 100 if no single stock > 5% of total portfolio
   - Score 70 if any stock 5–8%
   - Score 40 if any stock 8–12%
   - Score 0 if any stock > 12%

5. SECTOR CONCENTRATION SCORE (15% weight):
   - Score 100 if no sector > 30% of portfolio
   - Score 70 if any sector 30–40%
   - Score 30 if any sector > 40%

Return: overall score (0–100) and per-component breakdown.

Also generate red_flags (list of strings) — critical issues:
- "⚠️ 2 of your funds are Regular plans — costing you ₹X extra per year"
- "⚠️ Reliance Industries comprises 8.4% of your total portfolio — high single-stock risk"
- "⚠️ HDFC Flexi Cap and Mirae Asset Large Cap have 34% overlap — consider removing one"
- "⚠️ Your IT sector exposure is 42% — above recommended 30% threshold"

TASK 5: Portfolio Router

Create: backend/routers/portfolio.py

POST /api/portfolio/analyze
Request body:
{
  "funds": [
    {"name": "Parag Parikh Flexi Cap", "units": 150.5},
    {"name": "Mirae Asset Large Cap", "units": 80.0},
    {"name": "HDFC Mid Cap", "units": 200.0}
  ]
}

Process:
1. Resolve each fund name to a scheme_code (use fund_resolver service)
2. Fetch current NAV for each fund
3. Calculate current value = units × NAV
4. Fetch holdings for each fund from DB (with Redis cache)
5. Run OverlapEngine.compute_overlap()
6. Run OverlapEngine.compute_sector_exposure()
7. Run OverlapEngine.compute_stock_concentration()
8. Run OverlapEngine.compute_marketcap_breakdown()
9. Calculate expense audit
10. Run HealthScorer.calculate()
11. Run LLMExplainer.generate_portfolio_summary()
12. Save to analysis_sessions table
13. Return full AnalysisResult with session_id

Response: AnalysisResult schema

GET /api/portfolio/report/{session_id}

- Return same AnalysisResult
