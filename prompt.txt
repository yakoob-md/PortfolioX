TAX RULES TO IMPLEMENT (India, post-2024 Budget):
Equity Mutual Funds:
- STCG (Short Term Capital Gains): holding period < 12 months → taxed at 20%
- LTCG (Long Term Capital Gains): holding period ≥ 12 months
  → first ₹1,25,000 per financial year: EXEMPT
  → above ₹1,25,000: taxed at 12.5%

Debt Mutual Funds (post April 2023):
- Gains taxed as per income tax slab (added to income) regardless of holding period

ELSS Funds:
- Lock-in 3 years
- Qualifies for 80C deduction (up to ₹1.5L)
- LTCG rules apply after 3 years

FIFO (First In, First Out) Rule:
When units are redeemed, the oldest units are considered sold first.
This is mandatory per Indian tax law.

TASK 1: CAMS PDF Parser

Create: backend/services/pdf_parser.py

CAMS Consolidated Account Statement format:
- Issued by CAMS (Computer Age Management Services)
- Contains: investor name, PAN, folio number, scheme name, transaction history
- Each transaction: date, description (Purchase/Redemption/Dividend), units, NAV, amount

Transaction types to detect:
- Purchase (SIP/Lumpsum) → adds units with cost
- Redemption → removes units (triggers FIFO)
- Dividend Reinvestment → adds units
- Bonus → adds units at zero cost
- Switch In/Out → treat as redemption (out) + purchase (in)

class CAMSParser:
    
    def parse(self, pdf_path: str) -> ParsedStatement:
        """
        Returns:
        - investor_info: {name, pan (masked), email}
        - folios: list of Folio objects
        
        Folio:
        - scheme_name: str
        - scheme_code: str (look up from DB)
        - folio_number: str
        - current_units: float
        - transactions: List[Transaction]
        
        Transaction:
        - date: date
        - transaction_type: Literal['purchase', 'redemption', 'dividend', 'bonus', 'switch_in', 'switch_out']
        - units: float
        - nav: float
        - amount: float
        """

IMPORTANT PARSING NOTES:
- CAMS PDFs can have multiple formats depending on the year
- Use pdfplumber to extract text page by page
- Look for patterns like:
  - "HDFC BANK LIMITED" followed by fund entries → AMC section
  - Lines with dates in format "DD-Mon-YYYY" are transactions
  - "Purchase" and "Redemption" keywords identify transaction type
- Build format-detection: identify PDF version by looking for known header patterns
- Be fault-tolerant: if one folio fails to parse, continue with others

TASK 2: FIFO Engine

Create: backend/services/tax_engine.py

class TaxEngine:
    
    def calculate_gains(self, folios: List[Folio]) -> TaxCalculation:
        """
        For each folio:
        1. Build a purchase queue (FIFO) from all purchase transactions
        2. For each redemption:
           a. Pop from the front of the purchase queue
           b. Calculate holding period (redemption date - purchase date)
           c. If equity fund and holding < 12 months: STCG
           d. If equity fund and holding >= 12 months: LTCG
           e. Calculate gain = (redemption NAV - purchase NAV) * units
        3. Aggregate: total STCG, total LTCG
        4. Apply exemptions:
           - LTCG: ₹1,25,000 exemption per year
        5. Calculate tax:
           - STCG tax = STCG amount × 20%
           - LTCG tax = max(0, LTCG - 125000) × 12.5%
        """
    
    def calculate_tax_loss_harvesting(
        self, 
        folios: List[Folio],
        current_navs: Dict[str, float]
    ) -> List[HarvestingOpportunity]:
        """
        For each holding with unrealized losses:
        - If current NAV < purchase NAV → unrealized loss
        - If redeeming now would save taxes → flag it
        - Calculate: if you redeem X units at loss → offset Y in STCG gains → save Z in tax
        """
    
    def calculate_ltcg_optimization(
        self,
        folios: List[Folio]
    ) -> LTCGOptimization:
        """
        Check remaining LTCG exemption for the year (₹1.25L).
        Suggest: "You can realize ₹X more in LTCG this year tax-free.
        Consider redeeming some units in [specific fund] to utilize this."
        """

Tax calculation models:
class GainEntry(BaseModel):
    scheme_name: str
    purchase_date: date
    redemption_date: date
    holding_days: int
    gain_type: Literal['STCG', 'LTCG']
    purchase_nav: float
    redemption_nav: float
    units: float
    gain_amount: float
    tax_applicable: float

class TaxCalculation(BaseModel):
    financial_year: str  # e.g., "2024-25"
    total_stcg: float
    total_ltcg: float
    ltcg_exempt: float   # min(total_ltcg, 125000)
    ltcg_taxable: float  # max(0, total_ltcg - 125000)
    stcg_tax: float      # total_stcg * 0.20
    ltcg_tax: float      # ltcg_taxable * 0.125
    total_tax: float
    gain_entries: List[GainEntry]
    harvesting_opportunities: List[HarvestingOpportunity]
    ltcg_optimization: LTCGOptimization

TASK 3: Tax API Endpoints

POST /api/tax/upload
- Accepts multipart/form-data with PDF file
- Max file size: 10MB
- Process: parse PDF → calculate gains → return TaxCalculation
- Store parsed data in session (not permanently — privacy)
- Return: TaxCalculation + session_id for shareable link

POST /api/tax/manual
- For users who don't want to upload PDF
- Accepts manual transaction entries
- Same calculation logic as PDF upload

GET /api/tax/report/{session_id}
- Return stored tax calculation
- Expires after 24 hours

SECURITY FOR PDF UPLOAD:
- Validate file is actually a PDF (check magic bytes, not just extension)
- Process in memory — do NOT write to disk permanently
- Delete temp file immediately after parsing
- Never log file contents
- Add file size limit: 10MB max
